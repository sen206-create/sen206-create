## Portfolio Structure

Here is the kind of repository layout I aim for in my computational neuroscience projects:

```text
project-name/
├── README.md              # Project overview, motivation, figures, and usage
├── data/                  # Small sample data or download instructions
├── notebooks/             # Exploratory and narrative analysis notebooks
├── src/                   # Reusable package code
├── tests/                 # Unit tests for core analysis logic
├── figures/               # Generated plots and diagrams
├── environment.yml        # Reproducible environment
└── scripts/               # Command-line scripts for running analyses
```
